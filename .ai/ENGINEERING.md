# ENGINEERING.md — Padrões de Engenharia

## Código

- Funções pequenas.
- Classes com responsabilidade clara.
- Preferir composição a herança.
- Separar CLI de serviços.
- Evitar duplicação.

## Python

- pathlib para caminhos.
- sqlite3 ou camada de serviço.
- rich para UI terminal.
- pyyaml para configuração.
- pytest para testes.

## Commits

Formato sugerido:

- feat(audio): adiciona AudioService
- fix(database): corrige evento duplicado
- docs: atualiza roadmap
- refactor(kernel): simplifica inicialização
