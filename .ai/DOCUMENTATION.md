# DOCUMENTATION.md — Política de Documentação

## Documentos principais

- README.md: introdução e uso.
- MASTER_SPEC.md: especificação central.
- ARCHITECTURE.md: arquitetura técnica.
- ROADMAP.md: planejamento.
- TASKS.md: backlog.
- CHANGELOG.md: histórico.
- docs/adr/: decisões.

## Regras

Toda funcionalidade nova deve atualizar:

- README, se alterar uso;
- ARCHITECTURE, se alterar arquitetura;
- TASKS, se concluir tarefa;
- CHANGELOG, sempre;
- ADR, se houver decisão relevante.

## Diagramas

Preferir Mermaid para diagramas versionáveis.
