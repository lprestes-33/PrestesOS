# RELEASE.md — Processo de Release

## Versionamento

Usar versionamento semântico:

- MAJOR: quebra compatibilidade.
- MINOR: novas funcionalidades.
- PATCH: correções.

## Antes de release

1. Rodar testes.
2. Atualizar CHANGELOG.
3. Atualizar README.
4. Conferir ROADMAP.
5. Gerar tag Git.
6. Criar pacote ZIP.
7. Validar instalação limpa.

## Artefatos

- ZIP do código.
- Notas da versão.
- Documentação atualizada.
